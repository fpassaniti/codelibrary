Widget Tricks - ods-table: Fixed column

Fix 1, 2 or 3 columns New

By adding the CSS class sticky-col-1, sticky-col-2 or sticky-col-3 to an ods-table, you can respectively fix the 1, 2 or 3 first columns.

The width of the column, and the cell color are defined through CSS variables.

HTML CSS 
<ods-dataset-context context="ushospitals"
                     ushospitals-dataset="us-hospitals"
                     ushospitals-domain="userclub">
    <h3>1 column fixed</h3>
    <ods-table context="ushospitals"
               class="sticky-col-1"
               style="height: 220px;"></ods-table>
</ods-dataset-context>

<ods-dataset-context context="belgiumstatisticaldistricts"
                     belgiumstatisticaldistricts-dataset="belgium-statistical-districts"
                     belgiumstatisticaldistricts-domain="userclub">
    <h3>3 column fixed</h3>
    <ods-table context="belgiumstatisticaldistricts"
               class="sticky-col-3"
               style="height: 220px;"></ods-table>
</ods-dataset-context>
/** ODS-TABLE FIXED COLUMN **/

[class*="sticky-col-"] {
    --counter-col-width: 35px; /* Index column, record counter */
    --1st-col-width: 170px; /* First data column */
    --2nd-col-width: 150px; /* Second data column */
    --3rd-col-width: 100px; /* Third data column */

    --bg-color-odd: #f4f4f4;
    --bg-color-even: white;
    --hover-color: #e0e0e0;
}

[class*="sticky-col-"] thead.odswidget-table__internal-header-table-header,
[class*="sticky-col-"] table.odswidget-table__internal-table {
    position: relative;
}

Override variables to change column width and cell color

Column width:

  • --counter-col-width Set the record counter width
  • --1st-col-width Set the First data column width
  • --2nd-col-width Set the Second data column width
  • --3rd-col-width Set the Third data column width

Cell color:

  • --bg-color-odd Set the odd cell background color
  • --bg-color-even: Set the even cell background color
  • --hover-color: Set the hover cell color
HTML CSS 
<ods-dataset-context context="ushospitals"
                     ushospitals-dataset="us-hospitals"
                     ushospitals-domain="userclub">
    <h3>1 column fixed, larger first column, darker fixed cell</h3>
    <ods-table context="ushospitals"
               class="sticky-col-1 largercol darkerfixed"
               style="height: 220px;"></ods-table>
</ods-dataset-context>
<br/>

    <ods-table context="belgiumstatisticaldistricts"
               class="sticky-col-3 smallercol darkerfixed"
               style="height: 220px;"></ods-table>
</ods-dataset-context>

Responsive concern, unfix columns for smaller devices

Fixing too many columns can lead to a totally stuck and unreadable table on mobile device.

ods-get-element-layout get the size of an element and output the value in a variable. (Widget documentation here).

We can then fix the right number of columns depending on the width of the table.

Note: you can even fix columns only for desktop devices, and any for mobile

HTML CSS 
<ods-dataset-context context="belgiumstatisticaldistricts"
                     belgiumstatisticaldistricts-dataset="belgium-statistical-districts"
                     belgiumstatisticaldistricts-domain="userclub">
    <h3>3 column fixed for wide screen, 1 column fixed when the table is reduced</h3>
    <div ods-get-element-layout="tablelayout">
        <ods-table context="belgiumstatisticaldistricts"
                   class="darkerfixed"
                   ng-class="{'sticky-col-1': tablelayout.width < 800, 'sticky-col-3': tablelayout.width >= 800}"></ods-table>
    </div>
</ods-dataset-context>
